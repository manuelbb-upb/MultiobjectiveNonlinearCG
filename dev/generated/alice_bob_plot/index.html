Pareto-Optimality

Consider the formal optimization problem

\[\min_{\symbf{x}\in ‚Ñù^N} \begin{bmatrix} f_1(\symbf{x}) \\ \vdots \\ f_K(\symbf{x}) \end{bmatrix} \tag{MOP}\]

What are solutions of a problem with multiple objectives? Well, acceptable trade-offs between the objective function values. Formally, we use the concept of Pareto-optimality.

A point $\symbf{x}^*\in ‚Ñù^N$ is called Pareto-optimal if there is no $\symbf{x}\in ‚Ñù^N$

  • that is at least as good as $\symbf{x}^*$, i.e., no $\symbf{x}$ with $f_‚Ñì(\symbf{x}) ‚â§ f_‚Ñì(\symbf{x}^*)$ for all $‚Ñì=1,‚Ä¶,K$,
  • and that is strictly better in at least one objective, i.e., no $\symbf{x}$ for which there is some $‚Ñì\in\{1,‚Ä¶,K\}$ with $f_‚Ñì(\symbf{x})<f_‚Ñì(\symbf{x}^*)$.

The set of all optimal solutions is the Pareto-Set and its image is the Pareto-Front.

Example

Alice üë© and Bob üë® want to meet. Alice lives at $(1, 1)$ and Bob lives at $(-1, -1)$. So the distance to Alice's home is

\[d_A(x_1, x_2) = \sqrt{(x_1-1)^2 + (x_2-1)^2}.\]

For Bob, it is

\[d_B(x_1, x_2) = \sqrt{(x_1+1)^2 + (x_2+1)^2}.\]

Deciding on a meeting venue is the bi-objective optimization problem

\[\min_{\symbf{x}\in ‚Ñù2} \begin{bmatrix} d_A(\symbf x) \\ d_B(\symbf x) \end{bmatrix}\]

function setup_fig()

    ax1 = Axis(fig[2,1]; aspect=1.0)

To compare various meeting positions, we build a function testing for Pareto-optimality in a discrete set with value vectors Y.

function check_optimal(Y)
    _Y = empty(Y)               # array of processed value vectors
    isoptimal = zeros(Bool, 0)  # flags
    for fi in Y
        fi_isoptim = true
        for (j,fj) in enumerate(_Y)
            if all(fj .<= fi) && any(fj .< fi)
                # fj is better than fi
                fi_isoptim = false
                break
            end
            # test, if fi is better than fj
            if isoptimal[j]
                if all(fi .<= fj) && any( fi .< fj)
                    isoptimal[j] = false
                end
            end
        end
        push!(_Y, fi)
        isoptimal = vcat(isoptimal, fi_isoptim)
    end
    return isoptimal
end

With more points, it gets more complicated. We want to scatter the X and Y points and use different markers for optimal and non-optimal points.

function scatterOpt!(axX, axY, X, Y, isopt; colors)
    @assert length(X) == length(Y) == length(isopt)

    # plot non-optimal points first
    si = sortperm(isopt)

    for i=si
        color = colors[i]
        marker, strokewidth, strokecolor = if isopt[i]
            (:circle, 0.5, :black)
        else
            (:xcross, 1.5, :red)
        end

        scatter!(axX, (X[i]...); color, marker, strokewidth, strokecolor)
        scatter!(axY, (Y[i]...); color, marker, strokewidth, strokecolor)
    end
    nothing
end

That's it! With more samples we get a better sense for the optimal points of the continuous problem (MOP). Thus, let's use a pseudo-random sampling of decision space and see if we can identify the Pareto Set.

using HaltonSequences
using ColorSchemes
num_X = 200
X = [Tuple( -1.4 .+ 2.8 .* p  ) for p in HaltonPoint(2; length=num_X)]
Y = [(dA(x[1], x[2]), dB(x[1], x[2])) for x in X]
scatterOpt!(
    ax1, ax2, X, Y, check_optimal(Y);
    colors=[get(ColorSchemes.acton, (i-1)/(num_X-1)) for i=1:num_X]
)
fig

Ok, we got a rough sense for the Pareto Set. Indeed, we can analitically show that its the line from $(-1, -1)$ to $(1,1)$.

lines!(ax1, [(-1, -1), (1, 1)]; linewidth=5f0, color=:blue)
